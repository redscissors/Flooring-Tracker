# Test sheet analysis: Virginia Tile "EFT" price list (VTC_EFT_25_07_28.xls)

First real special-order sheet, provided 2026-07-12. Parsed with SheetJS
(`header:1` arrays, same as the app's import path). The 4 MB .xls itself is
NOT committed — it contains the shop's dealer costs and account name; this
analysis is the record.

## Shape

4 sheets: `Helper Sheet` (formula-flattened SKU/price/UM list), `Index`
(series → lead time, plus a **built-in "Markup %" cell** — the vendor designed
this template for dealers to apply their own markup), `MFG Data` (the real
data), `Comments & Key` (legend prose). Parse **MFG Data only**.

`MFG Data`: title/legend prose in rows 0-13, header row at index 14,
then 6,792 item rows:

| Col | Header | Maps to |
|---|---|---|
| 0 | *(none — status flag)* | see flag legend below |
| 1 | VTC MFG | manufacturer code (14 distinct: VTC 3411, FLO 809, ADX 635, CTI 451, …) |
| 2/3 | VTC Color / VTC Pattern | color/style |
| 4 | VTC Item Code | **sku** — 9-16 chars, alphanumeric, zero pure-digit |
| 5 | *(none)* | **description** ("EARTH ASH GRAY 3X12"; 5,904 of 6,792 contain a LxW size) |
| 6 | Product Line Name | series (150 distinct; top: PRESLEY 936, NANTUCKET 270, …) |
| 7 | VTC ESTIMATED LEAD TIME | READY SHIP 4,580 · IMPORT 2,189 · MADE TO ORDER 23 |
| 8 | CONSUMER LEVEL PRICE (Dealer to Consumer) | vendor's suggested retail (equal to dealer price everywhere in this file) |
| 9 | DEALER PRICE (VTC to Dealer) | **cost** — numeric on all 6,792 rows |
| 10 | Price U/M | SF 3,598 · PC 2,754 · SH 192 · EA 156 · ST 92 |
| 11 | No Broken U/M | smallest sellable unit |
| 12/13 | PC/CT · SF/CT | carton data — SF/CT numeric on 4,806 rows → carton math applies |
| 14 | Additional Comments | note ("DISCO BY ADX 6-2025") |

Flag legend (col 0, from the sheet's own KEY): `v` new · `◪` manufacturer
transitioning (249) · `x` phasing from stock · `xx` **discontinued** (169) ·
`•` made-to-order (23) · `*` **additional freight charges** (312 — mostly CER
48×48 slabs).

No duplicate item codes. No N/A prices in this account's file (the template's
key says N/A can appear — treat non-numeric cost as "no price", like the stock
parser's "See Catalyst" handling).

## What this confirms in the design

- **Per-book SKU pattern is essential**: the stock book's `/^\d{4,8}$/` would
  consume **zero** rows here. Pattern for this book: `/^[A-Z0-9]{9,16}$/i`.
- **Cost, not sell**: DEALER PRICE is the cost the markup applies to. The
  vendor's own template has a markup cell — the §3 model is exactly how this
  document expects to be used.
- **Carton math carries over**: SF/CT → `sfPerUnit`, U/M PC/EA → count lines,
  SF → sqft lines. Same `stockPatch` shapes.
- **Diff-preview honesty**: `xx` discontinued and comment column map straight
  onto the existing `discontinued`/`note` fields.

## What this breaks in the design (v1 assumptions)

1. **Scale: 6,792 items in ONE book** — ~10× the stock book. 30 books at this
   size ≈ 200k rows. Eager client-side loading of all books is dead; §6's
   "maybe load-on-first-search" hedge is resolved: **server-side search**
   (or per-book lazy load capped by measured size) is required from Phase 1.
   Version snapshots at ~1.5-2 MB jsonb per import are still fine.
2. **Markup groups ≠ sections.** This book has no "sections" in the stock
   sense; its natural group axes are **manufacturer (14 values)** or product
   line (150 — too many to hand-edit). The markup editor must key on a
   **group column chosen at mapping time**, not on the parser's section
   field. For VTC the sane choice is MFG.
3. **Freight is per-ITEM here, not only per-manufacturer**: the `*` flag marks
   312 items (oversized slabs). The book-level freight config stands, but
   items must carry a `freightFlag`, and a flagged pick must surface it
   (warning chip + the per-job freight line offer).
4. **Headerless columns are real**: the status flag (col 0) and description
   (col 5) have no header text. The mapping UI cannot be header-name-only —
   it must let the user label any column, and it needs a "flag legend" map
   (`xx` → discontinued, `*` → freight, `•` → made-to-order).
5. **Item fields to add**: `mfg` (group key), `leadTime` (a salesperson
   quoting IMPORT vs READY SHIP needs this at pick time — show it in search
   results and maybe as a chip on the selection), `msrp` (consumer-level
   price, reference only), `freightFlag`.
