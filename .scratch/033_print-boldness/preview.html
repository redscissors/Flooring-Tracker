Print boldness — estimate sheet options (issue 033)

The printed estimate reads light because most of its supporting text is mid-gray (--ft-muted #57534C, --ft-faint #8A8378) set in Manrope 400 at 10–11px, the ink is a soft near-black (#1C1A17), and the row rules are #CBC6B9 — all of which a grayscale laser dithers even lighter. Below: the current sheet next to three darkening treatments. Zoom to ~100% and squint, or print this page.

1 · Current

As deployed today.

2 · Darker ink

Colors only: true black ink, grays pulled two steps darker, rules darkened. Weights untouched — layout identical.

3 · Heavier type

Weights only: body 400→500, bolds 700→800. Colors untouched.

4 · Both — recommended

Darker ink + heavier type together. The biggest lift on paper while keeping the hierarchy (muted specs still read quieter than names).

What each option costs

Options 2 and 4's color changes are a token tweak in the .ft-light block of src/index.css (the estimate paper subtree) — no layout risk. The weight bump in 3/4 is one inherited font-weight:500 on the paper container plus changing the sheet's inline 700s to 800 in App.jsx; Manrope 500/800 are already loaded, and at these sizes the width change is negligible (no reflow/pagination risk seen at 780px). Both apply to the on-screen preview too, so preview stays WYSIWYG with the printout. The order sheet shares the same .ft-light wrapper and would darken along with it.

Also worth checking

If the printer is in toner-save / EconoMode, every option still prints light — worth ruling out on the shop printer. And the sand bands only print when "background graphics" is on in the print dialog (Chrome usually honors the app's print-color-adjust:exact, but other browsers may not).
